refactor(identity-access): consolidate admin and superadmin user management routes

This refactor unifies identity and access management workflows by consolidating
admin and superadmin user, student, instructor, and student whitelist management
under a single `/administrators` route group. It eliminates duplicate code between
the former `(protected)/(admin)/users` and `(protected)/(superadmin)/administrators`
paths, updates sidebar navigation labels and links, cleans up deprecated route
files, and standardizes shared components and hooks across both admin roles.

### Phase Progress

- Phase: 1 - UI Adjustment

### Tasks Done So Far

- renamed admin sidebar "Users" menu to "Identity & Access"
- migrated all superadmin administrator management code to top-level `administrators` route group
- consolidated duplicate user management components, hooks, and type definitions between admin and superadmin paths
- deleted deprecated `(admin)/users` and old superadmin administrator route files
- updated user management page imports and permission gate configurations
- updated workspace shell layout to pass user role for role-based navigation
