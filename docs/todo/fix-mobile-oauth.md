## Implemented Direction

[sentinel-mobile] owns its OAuth flow directly. It is a native React Native app running in Expo Go during development, so Google OAuth should return to the Expo app callback, not to [sentinel-web].

1. [sentinel-mobile] creates its native app callback URL with Expo Linking. In Expo Go this is typically an `exp://.../--/auth/callback` URL; in a development/standalone build this can be `sentinel-mobile://auth/callback`.
2. [sentinel-mobile] starts Supabase Google OAuth directly with `redirectTo` set to that native callback URL.
3. Supabase redirects Google OAuth back to [sentinel-mobile].
4. [sentinel-mobile] stores the session with `supabase.auth.setSession` and navigates to the classroom tab.

## Environment Variables

### sentinel-mobile

```env
EXPO_PUBLIC_MOBILE_AUTH_CALLBACK_PATH=auth/callback
EXPO_PUBLIC_EXPO_AUTH_PROXY_URL=
```

- `EXPO_PUBLIC_MOBILE_AUTH_CALLBACK_PATH` is optional. It defaults to `auth/callback`.
- `EXPO_PUBLIC_EXPO_AUTH_PROXY_URL` is optional. In Expo Go, the app defaults to `https://auth.expo.io/@livadomc/sentinel-mobile` and opens the proxy through its `/start` endpoint so the iOS auth browser can hand the result back to Expo Go reliably.

## Supabase Redirect URL Requirement

Supabase must allow the native callback URL generated by Expo. If Supabase does not allow the exact `redirectTo` value, it falls back to the project's Site URL, which is why the browser lands in production [sentinel-web].

Examples:

```text
https://auth.expo.io/@livadomc/sentinel-mobile
exp://192.168.1.102:8081/--/auth/callback
exp://localhost:8081/--/auth/callback
sentinel-mobile://auth/callback
```

For Expo Go, add the `https://auth.expo.io/@livadomc/sentinel-mobile` URL and confirm the generated `redirectUri` from the Metro console log is also present in Supabase Auth redirect URLs. For a production/dev-client build, prefer the stable custom scheme: `sentinel-mobile://auth/callback`.

After changing Expo env values, restart Expo with a cleared cache:

```bash
pnpm --dir app/sentinel-mobile start -- --clear
```
